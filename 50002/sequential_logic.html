Overview

Look at the diagram below.

\includepic{0.5}{Q1}

So far we have learned combinational device (also called combinational logic), that implements your truth table functional specs. But it isn’t enough to make a complete computer that we know today. We are missing the storage (memory) device. A simple combinational logic device does not remember its output value. It only gives an output when there’s an input, and the output(s) stay(s) stable for as long as its input(s) is/are stable. A memory device on the other hand, is a device of which we can \textit{write} new values to and is able to remember this value for a period of time, without the presence a stable input.

Storage Device: D-Latch

A D-latch is a mux with a feedback loop,

\includepic{0.5}{Q2}

\newpage

How it works:

  1. G is clock signal. It will periodically switch between '1’s and '0’s

  2. If G is 1, then the input signal on wire D will be passed to output wire Q, independent of the signal on wire Q’. Lets call this the write mode.

  3. If G is 0, then output signal on wire Q follows the signal on wire Q’, independent of input wire D. Lets call this the read memory mode.

Possible problems:

  1. Storing invalid information: If G changes from 1 to 0 at the exact moment when D is changing (invalid), then we will end up storing that new invalid information when the latch enters memory mode.

  2. Unstable output due to unstable input: In practice, input from users can be highly unstable. This will affect the output at Q as if G is 1, then it will pass all input from D to the output wire Q, regardless on whether it is a valid or stable input or not.

We address these problems in the next two sections.

The Dynamic Discipline

The dynamic discipline addresses problem 1 above: storing invalid information. It is very important to never violate the dynamic discipline to ensure any sequential logic circuits to work properly.

\\

Take a look at the clock signal below for G, it basically changes between 0 to 1 {periodically}:

\includepic{0.5}{Q3}

\newpage

The dynamic discipline states that:

  1. Tsetup=2tpdT_{setup} = 2 t_{pd}Tsetup​=2tpd​ of D-latch

  2. Thold=tpdT_{hold} = t_{pd}Thold​=tpd​ of D-latch

where,

  1. TsetupT_{setup}Tsetup​ = the minimum amount of time that the voltage on wire D needs to be stable before the clock edge changes from 1 to 0.

  2. TholdT_{hold}Thold​ = the minimum amount of time that the voltage on wire D needs to be stable after the clock edge changes from 1 to 0.

  3. tpdt_{pd}tpd​ is the propagation delay of the D-latch. Recall that D-latch for example, can be made of mux consisted of NAND gates, so the this tpdt_{pd}tpd​ is the propagation delay of that mux that is used to make up a D-latch - a combinational circuit.

Edge-Triggered Flip-Flop (Registers)

To address the second problem of ‘unstable output due to unstable input’, we put two D-Latches in series to make another device called the Flip Flop (also known as Registers),

\includepic{0.5}{Q4}

~\

The hardware of a Flip-flop:

  1. The first D-latch that receives the external input D is called the master latch, and the second D-latch is called the slave latch

  2. There is an inverter on the G input on the master flip flop, so the master latch “sees” the inverted clock signal

  3. The diagram on the right is how flip-flop symbol is drawn

How a flip-flop works:

  1. The user gets output only from the output wire of the slave Q wire, and the user supplies input only to the master D wire.

  2. Remember CLK periodically changes from 0 to 1 and vice versa

  3. When CLK signal is 0, the G wire of master latch will receive a 1 (due to the inverter) and the G wire of slave flip flop will receive a 0.

  4. This means that the master latch is in “write mode”, i.e: it lets signal from its D wire through to its Q wire, while the slave latch is in “read memory mode”, i.e: slave’s output depends on \textit{its own} memory wire Q’ and not affected by input on wire ⋆\star⋆.

  5. When CLK signal is 1, the G wire of master latch will receive a 0 due to the inverter and the G wire of slave latch will receive a 1.

  6. This means that the master latch is in “memory mode”, i.e: master’s output depends on \textit{its own} memory wire Q’ and not affected by input on wire D. Meanwhile, the slave latch is on “write mode”, i.e: it lets signal from the ⋆\star⋆ wire to be passed through its slave output.

  7. Hence, Only one of the two D-Latches is on “write mode” at a time or equivalently, only one D-latch is on “memory-mode” at a time.

The explanation above is illustrated in terms of waveforms below:

\includepic{0.5}{Q5}

The name “edge-triggered” comes from the fact that the output wire Q from the slave \textit{changes only when the CLK rises from 0 to 1}. Notice two things with this configuration:

  1. Unlike the \textit{star}, which is the output of one D-latch, Q signal is stable throughout an entire clock period, and change only in the next clock period. In comparison, the ‘star’ is only stable half the time when the master latch is at memory mode, but reflects ever-changing D-input signal during write mode.

  2. The edge-triggered flip-flop in this configuration, where the master is fed the INV CLK signal produces NEW value at Q at every rising edge of the CLK, as shown in Figure 5.

In the previous lessons, we learn about tCDt_{CD}tCD​ and tPDt_{PD}tPD​ for combinational device. For sequential device, i.e: circuits with CLK in it, these timings means:

  1. tCDt_{CD}tCD​ of a Register (or sequential logic unit with registers and CLs in it) is the time taken for invalid CLK input (not input to the sequential logic circuit) to produce an invalid output on wire Q (or the final output of the sequential logic unit)

  2. tPDt_{PD}tPD​ of a Register (or sequential logic unit with registers and CLs in it) is the time taken for valid CLK input (again, not input to the sequential logic circuit) to produce a valid output on wire Q (or the final output of the sequential logic unit)

Note the subtle difference between the tPDt_{PD}tPD​ and tCDt_{CD}tCD​ of a combinational vs a sequential device. In combinational device, there is no CLK involved, so tPDt_{PD}tPD​ of a combinational device is the time taken for a valid input to the circuit to produce a valid output of the circuit, and its tCDt_{CD}tCD​ is the time taken for an invalid input to the circuit to produce an invalid output of the circuit.

Flip-Flop Timing Constraint

For a single D-latch, we have the dynamic discipline to ensure that we do not end up storing invalid input signals. In the flip-flop configuration, we cascade two D-latches. Hence the dynamic discipline for the slave latch has to be obeyed by the master latch (not external input) The dynamic discipline is translated into this timing constraint for Flip-Flop:

[

t_{CD_{master}} > t_{hold_{slave}}

]

Explanation:

  1. When the INV CLK seen by master changes from 0 to 1, at the same time, the CLK signal seen by slave changes from 1 to 0.

  2. The master goes into write mode while the slave goes into memory mode.

  3. However, the ⋆\star⋆ at the output of the master cannot change (to follow new input at D, even though master is at write mode when INV CLK is at 1) in order to fulfil the thold{t_{hold}}thold​ requirement of the slave.

  4. This means the contamination delay of the master latch \footnote{Time taken on signal on ⋆\star⋆ is be invalid after G becomes invalid} has be larger than the hold time of the slave latch.

%1. What about the setup time of the slave latch? We do not have to worry about that because signal ⋆\star⋆ is taken care by the dynamic discipline (especially hold time) of the master latch \footnote{Recall dynamic discipline ensures that a latch will never end up going into read memory mode and storing invalid values}, meaning that ⋆\star⋆ is always stable before the clock edge changes from 0 to 1.

%

%1. Note: Some of you may realise that the sum of the hold time of the master latch and the setup time of the slave latch cannot be lesser than the clock period. This is true. But typically we will add combinational logic between two flip flops (see next section) so the clock period will always be much longer than the tholdmaster+tsetupslavet_{hold_{master}} + t_{setup_{slave}}tholdmaster​​+tsetupslave​​, hence we do not even discuss this case.

Sequential Logic Timing Constraint

We can now use Flip-Flop in our system as a ‘memory’ device that’s put in series, either before or/and after any combinational logic circuit. Due to dynamic discipline, we have two timing constraints called t1t_1t1​ and t2t_2t2​:

\includepic{0.5}{Q6}

Note: The ‘1’ in the picture above means any combinational logic unit, and it is unrelated with the ‘1’ in t1t_1t1​. The two boxes reg1reg_1reg1​ and reg2reg_2reg2​ are both flip flops, each, made up of 2 D-latches. tCLKt_{CLK}tCLK​ means the clock period.

\\

Explanation:

  1. The t1t_1t1​ constrains **ensures that the tHold requirement of the downstream register, R2, is fulfilled by the devices thats put before it, that is ‘1’ and R1. **

  2. The t2t_2t2​ constrains ensures that the tCLK (clk period) is enough for signals to propagate between two registers (R1 and R2), and also being set up at the downstream register R2.

  3. The t1t_1t1​ and t2t_2t2​ requirements must be fulfilled within any paths between two registers in a circuit, in order for the overall circuit to obey the dynamic discipline.

Looking at t2t_2t2​ constraint, we can deduce the time taken to do \textit{actual work or logic computation} (tPD1, or ‘loading time’) at the combinational logic unit ‘1’ is therefore,

[

t_{work_1} = t_{PD_1}= t_{CLK} - t_{setup_{R2}} - t_{pd_{R1}}

]

Synchronization

In any sequential logic circuit, we use a single synchronous clock, meaning that we use one same clock to any flip-flop in the circuit / device. Recall that dynamic discipline is crucial for any sequential logic circuit to work properly. However in practice, it is not possible for any arbitrary input to always be synchronised with the clock, i.e: to obey the tS and tH every cycle. We are now going to discuss what happens if dynamic discipline is violated.

\includepic{1.4}{metaplot.png}

Look at the figure above. The dynamic discipline ensures that the input D has to meet the setup and hold time of the register. But in reality, user does not always supply input that is synchronous with the clock edge. This carries the risk of violating dynamic discipline and we may end up storing the invalid values in read memory mode. This event of storing invalid value is called the metastable state, as shown in the figure above. For the entire clock cycle, the flipflop potentially produces an invalid output for an entire clock cycle, hence wasting that clock cycle as it is unable to give a valid output.

The Metastable State

Due to the feedback loop in the D-latch, the D-latch has a unique property where there exist a point in its voltage characteristics function whereby Vin = Vout. Take a look at the diagram in the figures below.

\includepic{0.3}{dff}

\includepic{0.7}{metastable}

The first figure signifies a D-latch, where its current CLK input is fixed at 0, so that the D-latch is at memory mode. Assume that we can measure Vin and Vout at the two points before and after the D-latch as shown in Fig. 9. In week 1, we learn how to plot the VTC, and the VTC of this D-latch is shown in green in Fig.10 below.

\\

The red line signifies the feedback constraint, where should we have Q at Vout equivalent to Q’ as Vin. This is the effect of connecting the output of the mux to itself. The green line signifies the VTC of a “closed latch” state. Notice that in the close latch state, the D-latch passes the value at Q’ as the output at Q, thus we have a shape that resembles that of a buffer. There are three solutions between the feedback constraint and the VTC of the closed latch, as indicated by the three circles in the figure: two end points that are stable, and one middle point that’s metastable (denoted as Vm).

~\

~\

What is the meaning this solution? Well, we are indeed creating a device which output is connected back as its input, so we need to know where the system will \textit{tend towards}.

~\

~\

In the figure above, if the initial \textit{Vin} value is below Vm, then it will tend towards the stable low indicated by the teal circle on the left. This is because at each loop, Vin is always greater than Vout, thus after a few loops, the value of Vout tends towards zero. The note at the figure above illustrates this scenario. Otherwise, the initial value of \textit{Vin} is above Vm, then it will tend towards the stable high indicated by the teal circle on the right, since Vin is always lesser than Vout at each loop.

~\

~\

However, if we have Vin = Vm, then the output will be again at Vm. This output of value Vm will be an input back at Vin, and without the presence of noise, if Vin is exactly at Vm then there’s a chance that we could wait forever for it to come to a stable value (basically the result never settles). A small presence of noise will drive Vin down or up and eventually it will settle to a stable value, however this is not guaranteed in bounded time.
