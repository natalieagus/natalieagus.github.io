Overview

The main aim for this chapter is to understand how we can utilise the topics we have learned in the previous chapters: pertaining combinational logic devices and synchronous sequential logic devices to create a finite state machine (FSM). We frequently use FSMs in our daily lives: traffic lights, vending machine, heating system, elevator, electronic locks (security systems), railroad systems, turnstile, alarm clock, and many more. We can create an FSM by implementing the functionality of the state machine using combinational logic devices, and assemble them with memory devices to form the complete (sequential logic) FSM circuitry:

Abstraction of Finite State Machine

A FSM (Finite State Machine) has:

  1. A set of kkk states: S1,S2,...,SkS_1, S_2, ..., S_kS1​,S2​,...,Sk​ (one of them should be the “initial” state)

  2. A set of mmm inputs: I1,I2,...,ImI_1, I_2, ..., I_mI1​,I2​,...,Im​

  3. A set of nnn outputs: O1,O2,...,OnO_1, O_2, ..., O_nO1​,O2​,...,On​

  4. Transition rules s′(Si,Ii)s'(S_i, I_i)s′(Si​,Ii​) for each of the kkk states and mmm inputs

  5. Output rules o(S)o(S)o(S) for each of the kkk states

A state machine is required when the same input produces multiple different outputs.

MOORE and MEALY Machine

There are two ways to draw the state transition diagram. The one shown in the previous section is called the Moore machine, meaning that the output is drawn on the states, meaning that the output depends on the CURRENT state. The second type is called the Mealy machine, where the output is drawn on the transition arcs. This means that the output depends on the CURRENT INPUT and state.

FSM Equivalence

\includepic{0.4}{Q4}

How to reduce FSM states:

  1. Two states SiS_iSi​ and SjS_jSj​ are identical iff

\begin{enumerate}

  1. States have identical output

  2. Every input ends (transit to) equivalent destination states (not just some of the inputs)

\item Find pairs of equivalent states, merge them

\end{enumerate}

Having less states will result in simpler functional specification table, and lesser bits to represent a state: comes at a cheaper cost (less registers used to store state bits) and smaller size.

Differences between Moore and Mealy Machine

Given an FSM, one can decide whether to build a physical Moore or Mealy machine. For a Moore machine,

  • Output obtained is depending only on the current state, meaning state and output production are synchronised.

  • Therefore, referring to Fig. 2, when one is at S011, and input is 0, the (next) output 1 is obtained at the same time when S0110 is outputted in the next clock cycle by the Register shown in Fig 8.

  • Sometimes, a Moore machine requires state. From Fig. 5, it is clear that the implementation using a Moore machine on the left requires 3 states, more than its corresponding Mealy machine.

  • A Moore machine comes with the benefit of synchronized output (at near the beginning of every clk cycle, we obtain the new state and the output of that state)

For a Mealy machine:

  • Output is obtained together with an input, meaning that they are reacting ‘instantly’ with the presence of an input.

  • Therefore, referring to Fig. 2, when one is at S011, and input is 0, the output 1 is obtained with response to the input, specifically one tpdt_{pd}tpd​ after input is fed to the logic device (refer to Fig. 9).

  • Note how we have lesser states in the Mealy machine, since we don’t combine the generation of state with the generation of output. This means that we may use less registers in a Mealy machine, potentially reducing its cost.

Final observation: A Mealy machine seems “faster” than the Moore machine since the output its gotten approximately tpdt_{pd}tpd​ after input arrives, while the output of a Moore machine is only obtained in the next CLK rising edge (notice tpdt_{pd}tpd​ of CL is lesser than CLK period).
